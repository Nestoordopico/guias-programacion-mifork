# TEMA 1. Clases y objetos

## 1. ¿Cuáles son las cuatro características básicas de la programación orientada a objetos? Describe brevemente cada una

### Las cuatro características fundamentales de la programación orientada a objetos forman el núcleo de este paradigma. Cada una aporta una forma distinta de organizar y estructurar el código para hacerlo más modular, reutilizable y fácil de mantener.

### 1. **Abstracción**
- Consiste en **simplificar la realidad**, modelando solo los aspectos esenciales de un objeto.
- Permite centrarse en *qué hace* un objeto, sin necesidad de conocer *cómo lo hace* internamente.
- Ejemplo: una clase *Coche* expone métodos como `arrancar()` sin mostrar la complejidad del motor.

 2. **Encapsulamiento**
- Agrupa datos (atributos) y comportamientos (métodos) dentro de una misma unidad: la clase.
- Protege la información interna mediante niveles de acceso (público, privado, protegido).
- Evita que el estado del objeto sea modificado de forma incorrecta desde fuera.

 3. **Herencia**
- Permite crear nuevas clases basadas en otras ya existentes.
- La clase hija **hereda** atributos y métodos de la clase padre, pudiendo añadir o modificar funcionalidades.
- Favorece la reutilización de código y la creación de jerarquías lógicas.

 4. **Polimorfismo**
- Habilita que un mismo método pueda comportarse de distintas maneras según el objeto que lo invoque.
- Puede manifestarse como *sobrecarga* (mismo nombre, diferentes parámetros) o *sobreescritura* (una clase hija redefine un método de la clase padre).
- Facilita escribir código más flexible y extensible.

## 3. Los paradigmas anteriores a la POO, ¿Qué es la **programación estructurada**? y, todavía mejor, ¿Qué es la **programación modular**?

### La programación estructurada organiza el código usando secuencia, decisiones e iteraciones para que el programa sea claro y fácil de seguir, evitando saltos desordenados como goto. La programación modular divide el programa en partes independientes llamadas módulos, cada uno con una función específica, lo que facilita entender, mantener y reutilizar el código.

## 4. ¿Qué tres elementos definen a un objeto en programación orientada a objetos?

### 1. Atributos
Son las características o datos que describen al objeto.
Ejemplo: en un objeto Coche, los atributos pueden ser color, marca o velocidad.

2. Métodos
Son las acciones o comportamientos que el objeto puede realizar.
Ejemplo: acelerar(), frenar(), encender().

3. Estado
Es la situación actual del objeto, determinada por los valores de sus atributos en un momento dado.
Ejemplo: un coche con color rojo, velocidad 80 y motor encendido.

## 5. ¿Qué es una clase? ¿Es lo mismo que un objeto? ¿Qué es una instancia? ¿Todos los lenguajes orientados a objetos manejan el concepto de clase?

### Clase: Es una plantilla que define qué datos y comportamientos tendrán ciertos objetos.
    Objeto: Es algo concreto creado a partir de una clase.
    Instancia: Es simplemente un objeto creado desde una clase; ambos términos suelen significar lo mismo.
    ¿Todos los lenguajes orientados a objetos usan clases? No. La mayoría sí, pero algunos usan otros modelos (por ejemplo, basados en prototipos).


## 6. ¿Dónde se almacenan en memoria los objetos? ¿Es igual en todos los lenguajes? ¿Qué es la **recolección de basura**? 

### ¿Dónde se almacenan los objetos?
Depende del lenguaje: algunos los ponen en pila o montón (como C++), otros casi siempre en el montón (Java, Python, C#).
    ¿Es igual en todos los lenguajes?
No. Cada lenguaje gestiona la memoria de forma distinta.
    ¿Qué es la recolección de basura?
Un sistema que borra automáticamente los objetos que ya no se usan para liberar memoria.
(Lo tienen Java, Python, C#, pero no C++).


## 7. ¿Qué es un método? ¿Qué es la **sobrecarga de métodos**? 

### Método:
Es una función que pertenece a una clase y describe una acción o comportamiento de sus objetos.
    Sobrecarga de métodos:
Es tener varios métodos con el mismo nombre, pero con parámetros diferentes.
El lenguaje elige cuál usar según los argumentos.


## 8. Ejemplo mínimo de clase en Java, que se llame Punto, con dos atributos, x e y, con un método que se llame `calculaDistanciaAOrigen`, que calcule la distancia a la posición 0,0. Por sencillez, los atributos deben tener visibilidad por defecto. Crea además un ejemplo de uso con una instancia y uso del método

### Aquí tienes un ejemplo mínimo en Java:
class Punto {
    int x;  // visibilidad por defecto
    int y;  // visibilidad por defecto

    double calculaDistanciaAOrigen() {
        return Math.sqrt(x * x + y * y);
    }
}
    Ejemplo de uso:
public class Main {
    public static void main(String[] args) {
        Punto p = new Punto();
        p.x = 3;
        p.y = 4;

        double d = p.calculaDistanciaAOrigen();
        System.out.println("Distancia al origen: " + d);
    }
}


## 9. ¿Cuál es el punto de entrada en un programa en Java? ¿Qué es `static` y para qué vale? ¿Sólo se emplea para ese método `main`? ¿Para qué se combina con `final`?

### Punto de entrada en Java: el método main, donde empieza la ejecución del programa.
    static: indica que algo pertenece a la clase, no a los objetos; puede usarse sin crear instancias.
    ¿Sólo para main? No. Se usa también para métodos y atributos compartidos.

## 10. Intenta ejecutar un poco de Java de forma básica, con los comandos `javac` y `java`. ¿Cómo podemos compilar el programa y ejecutarlo desde linea de comandos? ¿Java es compilado? ¿Qué es la **máquina virtual**? ¿Qué es el *byte-code* y los ficheros `.class`?

### Compilar: con javac, que convierte el código .java en bytecode (.class).
    Ejecutar: con java, que carga los .class y los ejecuta en la máquina virtual.
    ¿Java es compilado? Sí, pero se compila a bytecode, no a código máquina.
    Máquina virtual: la JVM, que interpreta/ejecuta el bytecode y lo hace portable entre sistemas.
    Bytecode y .class: el bytecode es el código intermedio generado por javac; los .class son los archivos que lo contienen.


## 11. En el código anterior de la clase `Punto` ¿Qué es `new`? ¿Qué es un **constructor**? Pon un ejemplo de constructor en una clase `Empleado` que tenga DNI, nombre y apellidos

### ¿Qué es new?
Es el operador que crea un objeto en memoria (en el heap) e invoca su constructor.
    ¿Qué es un constructor?
Es un método especial de la clase que inicializa el objeto al crearse.
Tiene el mismo nombre que la clase y no devuelve valor.
    Ejemplo de constructor en Empleado (DNI, nombre, apellidos)
class Empleado {
    String dni;
    String nombre;
    String apellidos;

    // Constructor por parámetros
    Empleado(String dni, String nombre, String apellidos) {
        this.dni = dni;
        this.nombre = nombre;
        this.apellidos = apellidos;
    }

    // (Opcional) Constructor por defecto
    Empleado() { }
}
